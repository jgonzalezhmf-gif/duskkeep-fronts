# Notas de Derechos de Assets

Fecha de corte: 2026-04-28

Este documento no es asesoramiento legal. Es una guia operativa para reducir riesgo antes de publicar o monetizar Duskkeep Fronts.

## Regla Operativa

Los assets visuales del juego deben ser:

- Generados para Duskkeep Fronts o creados internamente.
- Recortados/ajustados localmente sin incorporar material de terceros.
- Sin logos, personajes reconocibles, marcas, nombres o elementos protegidos de terceros.
- Registrados mediante manifests antes de cargarse en UI.
- Revisados visualmente antes de publicacion.

## Assets Generados con OpenAI

Segun los terminos de OpenAI, el usuario conserva derechos sobre el input y, en la medida permitida por la ley aplicable, posee el output. OpenAI tambien indica que no reclama copyright sobre contenido generado por la API para el usuario o sus usuarios finales.

Puntos practicos:

- Se pueden usar outputs en proyectos comerciales si se cumplen los terminos aplicables y la ley.
- La responsabilidad de revisar que el output no infringe derechos de terceros sigue siendo nuestra.
- El output puede no ser unico; otro usuario podria recibir algo similar.
- Evitar prompts que pidan copiar personajes, marcas, estilos extremadamente identificables o assets de terceros.

## Checklist Antes de Publicar Assets

1. Confirmar que el asset final no contiene marcas, logos o personajes reconocibles de terceros.
2. Confirmar que el prompt o flujo de generacion no dependia de una IP externa concreta.
3. Subir solo PNGs/WebP finales recortados, no laminas fuente ni pruebas.
4. Registrar el asset en el manifest correspondiente.
5. Verificar que no hay 404 ni rutas especulativas.
6. Si un asset va a ser central para marketing, store pages o monetizacion, hacer revision legal/humana especifica antes de publicarlo.
